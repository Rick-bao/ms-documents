


2015年9月7日：

代码规范：

1.代码注释，增强可读性

2.代码分层，即controller不处理业务数据，service不直接操作数据库，dao不做业务处理。

3.代码命名，避免使用可读性差的标识，如view1，result2等类似属性。

4.所有URL均采用绝对路径，避免资源移动导致错误信息出现

5.已经完善的代码在后期版本不做多次修改，全部通过新建功能代码实现模块更新。



2015年9月7日

数据库说明：

1.后台所有数据均作为历史查账，主运营库

2.后台数据定期备份至硬盘，作为账单依据

3.前端商品模块数据来自于后台处理后的数据信息，只作为显示给用户的信息

4.前端订单模块数据同样来自于后台处理后的数据，具体为：
	前端下单 -> 进入后台 -> 人工处理 -> 放入前端 -> 状态变更 -> 存入后台(已完成)

5.前端会员模块作为持久性数据保存，流程为：注册 -> 后台 -> 审核 -> 前端(通过)




数据库字段总体说明：

1. ID	char（32）	是			唯一标识

2. department_id	char(20)		单位、部门编号

3. parent	char(32)		是	单位：自关联

4. remarks	varchar(50)			说明

5. title	varchar(50)			标题

6. enable	char(2)				有效标识:有效、无效

7. content	varchar(100)			内容

8. phone_number	char(12)			电话：11-12位

9. recording_time	datetime		记录时间
